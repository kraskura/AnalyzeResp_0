Quality Control and Finalising data: a possible approach

Source: AnalyzeResp_0/blob/main/docs/QC_protocol.md

Data Analysis Pipeline

  1. Visualize Data:
    • Plot all MO2 data using the AnalyzeResp_0 code (MMR and SMR functions only at this point)
    • Visually inspect all MO2 plots (list of key plots to inspect below)
      • /AUTO/plots_channel
      • AUTO/plots_channel_cycle (when flush = TRUE)
      • AUTO/plots_channel_temperature_cycle
      • /MANUAL/channel_plots
  2. Group and Clean Data
    • Identify all clean MO2 data with linear decline - keep
    • Identify any plots with minor issues - correct
    • Identify erroneous MO2 plots - remove
  3. Background
    • Assess if background is present (SMR function run for background respiration)
    • If background is present, correct all MO2 slopes for background respiration
      • See Rosewarne et al 2016 for guidance
      • To customize the analysis use MMR_SMR_AS_EPOC function arguments: background_prior = NA, background_post = NA, background_slope = NULL, background.V = NULL, background_linear_gr = FALSE, match_background_Ch = FALSE, mmr_background = "SAME_slope"
    • We generally assume a linear increase between the “before” and “after” background and apply that to the slopes.
      • Use background_linear_gr = TRUE
  4. Plot the data and calculate MR values
    • Plot the (clean) data. Plot MO2 vs time and temperature vs time. ESSENTIAL for every fish.
    • Calculate SMR, RMR, MMR, EPOC, time to 50% MMR (Time[MMR50]) or 75% MMR (Time[MMR75])
    • Many ways to do this - talk to collogues and peers (i.e. which sliding measurement window for MMR; which SMR value; which EPOC threshold)
  5. Assess metabolic scaling
    • Plot the data - SMR vs body mass; MMR vs body mass (use log-log plots and raw data plots)
    • Adjust for scaling if needed (see above).
      • To customize the analysis use MMR_SMR_AS_EPOC function arguments: scaling_exponent_mmr=1, scaling_exponent_smr=1, common_mass=1

Additional notes and suggestions from the Eliason Lab

  1. Check for leaks
    • Plot the full measurement cycle at least once for a few respirometry trial runs:
      • SMR( ..., flush_plot = "ON")
    • Are ‘L’ shapes for measurement cycles (measure phase) consistent across files? If yes, there might be a leak.
  2. Notes and suggestions:
    • Do not cut the recovery or resting metabolic rate measurement down to below the “sliding window” length selected for MMR.
    • For cleaning: measurement cycles with R2 below 0.7 are immediately rejected: remove / no cleaning
    • Experimental notes: If there are any measurements with a complementary note that anyone was messing with the respirometry chamber (relieving a bubble, etc) or where data is missing from the measurement (low signal in probe can cause this), it is aadvised to exclude the measurement.
