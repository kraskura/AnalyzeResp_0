Understanding system-wide background respiration rates

Source: kraskura/AnalyzeResp_0

sum_backgr

About

  • Arguments
  • Value
  • Details
  • Example
  •   See datafiles on GitHub

Description

This function summarises many background files, best applied to summarise background that has been run for longer time (many measurement cycles). It becomes useful when trying to better understand system-wide respiration rate, across all channels. For example, it may be used to explore the variability of background respiration rates between different respirometry chambers,or to explore overtime changes in backgrounds respiration. This is shorthand way to get background MO2 mgO2/L/min through MMR_SMR_AS_EPOC function. Note: this is a developing function

Arguments

Argument Description Default
data Data file _…analyzed.csv of respiration. –
Ch Specify Channels for analysis. A numeric vector. –
resp.ID A character vector with respiromtery IDs. –
resp.Vol A numeric vector of respirometry volumes. –

Value

  Data outputs
  1. .csv files
  • ()_BACK_SLOPES.csv
    • Saved in local directory
    • Metrics included: Channel ID, temperature, R2 (min, max, mean), slopes (min, max, mean), the number of measurements, the ID and volume of respirometer, and the datafile name.
  1. .png files
  • ()_ChX_backSlopes_PLOT.png (X = the number of channel)
    • All files saved in the local directory
    • Frequency plot of all background respiration slopes for each channel.

Details


One file is analysed at the time, writing a loop to analyse all files in a given directory will be handy.


Example 


 

 sum_backgr(data = "jul04_2019_box4_back_post_dummy_analyzed.csv",
            Ch = c(1,2,3,4),
            resp.ID = c("A", "B", "D", "E"),
            resp.Vol = c(1.5, 1.5, 1.5, 1.5))
