Setup and data format

Are the data in correct format?

Can this be run on 2-channel Firesting?

  • This code was build using 4-channel FireSting files. For all 4 variable vectors, it still should work for 2- channel FireSting. Just indicate the missing data correctly.

Cannot find the converted .txt –> .csv files

  • If using organize_MR_analysis for organization, be sure to have all .txt files in the same directory as the all main folders (i.e. AUTO, MANUAL, MMR_SMR_AS_EPOC) , otherwise the code will ot be able to find the specific csv_files subfolder.

What R2 threshold to set?

  • If it is too high, all measurement cycle data may be discarded leaving you with no data.

Are you in correct working directory?

Leaky respirometry chambers?

Is water hypoxic? how to identify this?

  • use sodium sulfite or N2 gas to deoxygenate water and then run that in closed phase to see if the O2 gets in

Background respiration

Have a blank respirometer running aside with different volume (L) than the one with an animal. How to account for it?

  • In function MMR_SMR_AS_EPOC: background_slope and background.V must be provided together. The function will fail having only one without the other

Have one background measurement for each trail, how to add it to the function?

I have a background respiration estimates already, can I use it without giving a raw csv file?

  • Absolutley, In function MMR_SMR_AS_EPOC: if using background_linear_gr = TRUE, both background_prior and background_post must be provided
